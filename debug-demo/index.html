STEP 2.1

Syntax Errors Identified

❌ Snippet 1 — Missing parenthesis & mismatched quotes

function greetUser(name { // ← missing ')' in parameters console.log("Hello, " + name) // ← mismatched parenthesis } greetUser("Alice" // ← missing closing ')'

✔ Fixed

function greetUser(name) { return "Hello, " + name + "!"; }

❌ Snippet 2 — Mismatched quotes & missing closing brace

if (age >= 18) { console.log('You are an adult."); // ← mismatched ' and " else { // ← missing closing '}' before else console.log("You are a minor."); }

✔ Fixed

if (age >= 18) { return "You are an adult."; } else { return "You are a minor."; }

❌ Snippet 3 — Assignment (=) used instead of comparison (===)

if (x = 10) { ... } // ← '=' assigns; should be '==='

✔ Fixed

if (x === 10) { ... } // ← strict equality comparison
STEP 2.2

Runtime Errors Identified

❌ Snippet 4 — ReferenceError: Typo in function name

let result = mutiply(4, 5); // ← 'multiply' misspelled → ReferenceError

✔ Fixed

function multiply(a, b) { return a * b; } let result = multiply(4, 5); // ← correct name, declared before use

❌ Snippet 5 — Division by zero (no guard)

function divide(a, b) { return a / b; // ← returns Infinity when b === 0 }

✔ Fixed

function divide(a, b) { if (b === 0) return "Error: Cannot divide by zero."; return a / b; }

❌ Snippet 6 — TypeError: Reading property of null

let user = null; console.log(user.name); // ← TypeError: Cannot read properties of null

✔ Fixed

function getUserName(user) { if (user == null) return "Error: User not found."; return user.name; }

❌ Snippet 7 — Infinite loop (missing increment)

let i = 0; while (i < 5) { console.log(i); // ← 'i' never changes → infinite loop }

✔ Fixed

while (i < 5) { results.push(i); i++; // ← increment prevents infinite loop }
STEP 4.1

Debugging Techniques Used

  • › console.log() — Used to print variable values and function outputs at key points to trace the program's state during execution.
  • › Browser DevTools Console (F12) — Displays runtime errors with file name, line number, and error type (e.g., ReferenceError, TypeError).
  • › Sources → Breakpoints — Pause execution at specific lines to inspect variable values in the Scope panel and step through logic.
  • › Watch Expressions — Monitor specific variables continuously as code steps forward.
  • › Code Review — Carefully read each snippet to spot mismatched brackets, wrong operators, and undefined references before running.
